Problem Solving

"When the Code Gets Tough: Navigating Problem Solving as a Developer"

As a developer, we often come across challenges that require us to think creatively and critically to solve problems. In this blog post, I'll be sharing personal anecdotes of times when I faced challenges while coding and how I went about solving them.

The Frustration of Style Struggles

One time, I was working on a project and was struggling to style items according to screen width. I tried explaining my thought process to myself, which is a technique I like to call the "Rubber Ducky Method." I thought it should work, but it just wasn't happening. Eventually, I realized that I was using the code in the wrong way and needed to do further research to fix the issue. I learned about using @media in CSS and how it allows you to specify a media query and a block of CSS to apply to the document only if the media query matches the device on which the content is being used.

The Triumph of JavaScript Katas

Another time, I was working on some JavaScript Katas, which are coding exercises that help you improve your skills. I was a little frustrated because the questions were specific and not the generic code that most learning sites offer. However, I didn't give up. I used the "Research and Reflect" technique and reflected on past material, researching missing areas. I also tried explaining the information out loud, which helped me to simplify my understanding. In the end, I was able to navigate and solve the kata questions.

My Confidence in Problem-Solving Techniques

I feel confident in all problem-solving techniques and processes, even if I don't use them all. My go-to methods are Googling and reflection, and I feel very confident in these. The "Rubber Ducky Method" is also a close second. Reading error messages is also incredibly useful when trying to solve code. In conclusion, as a developer, you will face challenges, but with persistence and a willingness to learn, you can overcome them. Experiment with different problem-solving techniques and find what works best for you. Happy coding!
